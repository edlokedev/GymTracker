## Proposed CLAUDE.md Additions

- [ ] **Cron stacking: now 4× — urgently needs dedup** — The nightly review prompt enqueued 4 times simultaneously today (was 3× last time, 1× when first introduced). Each session adds a new CronCreate without removing the previous one. Since cronsare session-only, they don't stack within one session — the accumulation happens when the startup hook fires and the user doesn't close/reopen the session between days. Fix in the startup hook: emit `CronCreate` only once per session, not once per incoming message. Document in `nightly-review.md`: "Do not call CronCreate multiple times per session — the hook fires once on session start, not once per message."
- [ ] **`/tmp` nuance: works in bash, not in node.js paths** — `/tmp` in git-bash = `$USERPROFILE/AppData/Local/Temp` (same directory — confirmed by `cp: same file` error). But `node -e "fs.readFileSync('/tmp/file')"` resolves `/tmp` as `C:\tmp` (Windows path), which doesn't exist. Update CLAUDE.md Shell section: "`/tmp` is valid in bash commands. In node.js `fs` calls, use `process.env.USERPROFILE + '/AppData/Local/Temp/'` instead of `/tmp/`." The current "No /tmp" rule is overly broad — it's bash-safe, just node-unsafe.
- [ ] **`tail | python3 -c "..."` via RTK panics** — `tail ... | python3 -c "..."` caused a Rust panic in RTK ("failed printing to stdout: The pipe has been ended.") in multiple sessions (2026-05-28, 2026-06-01). RTK wraps the python3 binary and panics when the pipe closes early. Pattern to avoid: `cmd | python3`. Use `node -e` instead, or write to a temp file first.
